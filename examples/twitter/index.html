When either the first or last name changes, the display of the name updates. 
{
    selector: '[name=first-name]',
    observes: 'firstName',
    event: 'keyup',
    interface: 'value'
}, {
    selector: '[name=last-name]',
    event: 'keyup',
    observes: 'lastName',
    interface: 'value',
}, {
    selector: '#name'
    observes: ['firstName', 'lastName']
    interface: 'html'
    handler: 'getFullName'
}
When any of the editable fields change, the date updates 
{
    selector: '.date'
    observes: ['firstName', 'lastName', 'onTwitter']
    interface: 'text'
    handler: 'getDate'
}
If the first name is empty, the "On Twitter?" checkbox becomes disabled. 
{
    selector: '[type=checkbox]'
    observes: 'firstName'
    interface: 'enabled'
}
When the checkbox is checked, the "is on Twitter" is appended to the name display. 
{
    selector: 'input[type=checkbox]'
    event: 'change'
    observes: 'onTwitter'
    interface: 'checked'
}, {
    selector: '#onTwitter'
    observes: 'onTwitter'
    interface: 'visible'
}

Two-Way Binding (sync)

A two-way binding represents two objects that keep in sync with each no matter the source of the change. That is, if the element triggers the event to change, the model will update itself to reflect that change and vice versa.

Requires: selector event observes interface

One-Way Binding (handler)

A one-way binding using a handler (no model attributes being obvserved) has the view (or any object) observing an event from an actionable element such as a button or link. The handler is used to perform some processing indirectly of the model data.

Note, this is no different than a typical bind using jQuery e.g. $('#mybutton').bind('click', function() { alert('hi')' });. The one difference is that the selector is within the context of the view the bindings are defined on.

Requires: selector event handler

One-Way Binding (read-only)

A one-way binding that is read-only (ro) is typically used with an element that is non-actionable. It represents model data in some way and will update itself if the model data changes. There is no way for the user to alter that data using that element.

Requires: selector observes interface

One-Way Binding (write-only)

A one-way binding that is write-only (wo) is the opposite of the ready-only binding. The element provides a means of entering data using form fields and it will write to the model, but any changes to the model will not be reflected in the representative element.

Requires: selector event observes interface loopback=false
